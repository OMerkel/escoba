# Escoba de Quince - HTML5/JavaScript Implementation

Modern web-based implementation of the classic Spanish card game **Escoba de
Quince** (Sweep of Fifteen), featuring AI opponents using Greedy, Negamax, and
MCTS strategies.

### Development

```bash
npm run dev
```

Starts local dev server at `http://localhost:3000` with hot module reloading.

### Serving `src` vs `dist` (Live Server)

- `src/index.html` can work on VS Code Live Server in this project because it
  uses browser-native ES modules and relative CSS/JS links.
- `dist/index.html` is the production artifact generated by Vite and is the
  recommended deployment target.
- Vite is configured with `base: "./"` in `vite.config.js`, so built assets in
  `dist/index.html` use relative paths (`./assets/...`). This makes `dist`
  work when served from folder-based static servers such as VS Code Live
  Server.
- Static assets are kept under `src/img/` and copied to `dist/img/` at build
  time.

### Progressive Web App (PWA)

This app supports installation and offline play-oriented caching.

- **Manifest**: `src/manifest.json`
- **Service worker**: `src/sw.js`
- **Install UX**: browser-managed install flow (for example, Add to Home Screen
  on mobile or Install App from browser chrome on desktop)
- **Policy**: intentionally no in-app install button, hint, or custom install
  prompt handling

#### What Is Cached

- App shell assets (entry page, manifest, and key icons)
- Runtime same-origin assets (scripts, styles, images, workers, fonts) via
  stale-while-revalidate
- Navigation requests use network-first with offline fallback to cached
  `index.html`

#### How To Test Installation

1. Run `npm run build`.
2. Serve `dist/` with HTTPS or `localhost` semantics.
3. Open the app in a supported browser.
4. Install it using the browser's built-in install or Add to Home Screen UI.

#### How To Test Offline Behavior

1. Open the app once while online.
2. In browser devtools, switch to Offline network mode.
3. Reload the page; previously fetched app assets should continue to load from
   cache.

## 🎮 Game Rules

### Objective

Players score points by capturing cards, making *escobas* (sweeps), and winning
category-based bonuses. The first player to reach **21 points with at least a
2-point lead** wins.

**Winning Condition (Escoba de Quince Standard):**

- Player must reach ≥21 points (configurable: 11, 15, 21, or 30)
- **AND** maintain a minimum 2-point lead over opponent
- Examples:
  - 21–19: Player 1 wins ✓ (≥21 AND 2-point lead)
  - 21–20: Game continues (only 1-point lead)
  - 21–21: Game continues (tied)
  - 22–20: Player 1 wins ✓

### Card Values

| Card | Value |
| --- | ---: |
| As (Ace) | 1 |
| 2–7 | Face value |
| Sota | 8 |
| Caballo | 9 |
| Rey (King) | 10 |

### Scoring Categories (per round)

1. **Most Cards**: +1 point (cards captured)
2. **Most Oros** (coins): +1 point
3. **Seven of Oros**: +1 point (automatic)
4. **Setenta (Prime)**: +1 point (best card per suit; variant-dependent)
5. **Escobas**: +1 point each (clear all table cards in one capture)

### Capture Mechanics

- Play a card from hand
- Select table cards that sum to exactly 15 (with played card) to capture them
- **Preference rule**: Single-card captures (complement to 15) take precedence
  over multi-card captures
- **Escoba**: If all table cards are captured, score 1 escoba point

**UI Interaction:**

- Hand cards are clickable and show selection state
- Table cards are also clickable for multi-card capture combinations
- Instructions displayed: "Click table cards to capture with your selected hand
  card"
- Selected card count shown in control panel
- "Play Move" button executes capture; "Clear" resets selection

### Turn Sequence

1. **Deal**: 3 cards to each player, 4 to table (opening deal); 3 cards each on
   re-deals
2. **Play**: Player plays 1 card per turn
   - Capture valid combinations totaling 15
   - OR discard if no valid capture exists
3. **Redeal**: When both players exhaust their 3 cards, dealer redeals 3 more
   (no new table cards)
4. **Round End**: Stock exhausted; final table cards awarded to last capturer
5. **Score**: Calculate round points; check win condition
6. **Next Round**: Start new deal with dealer role rotated (if game continues)

### Configuration

```javascript
// Target scores (configurable)
const targetScores = [11, 15, 21, 30];  // Default: 21

// Setenta methods (scoring variants)
const setentaMethods = [
  "prime",        // Best card per suit (ranked 7→6→5→...)
  "simplified",   // Count of 7s
  "numerical"     // Numerical values (7=21, 6=18, As=16, ...)
];
```

For full rules, see [doc/rules.md](../../doc/rules.md) and
[doc/regeln.md](../../doc/regeln.md) (German).

## 🎨 UI Features

### Capture Display Visualization (FR-UI-1)

Each capture or discard is displayed for 4 seconds before execution, showing:

- **Played card** with value
- **Captured table cards** (if any) with values
- **Calculation sum** (e.g., "5 + 10 = 15")
- **Player label** (Human or AI) - helps distinguish moves in all game modes
- **Escoba indicator** 🧹 when sweep occurs

This feature helps all participants follow the gameplay, especially in these
modes:

- **AI vs AI**: Both moves are visible without the AI cards being shown
- **Human vs Human**: Non-active player's cards remain hidden while their move
  is displayed
- **Human vs AI**: AI moves are no longer "invisible" - all captures are clear

The display duration is configurable in source configuration via
`src/config/configuration.js`:

```json
{
export const DEFAULT_CONFIG = {
  // ...
  aiResponseTime: 5000,
  // ...
};
}
```

**Game Input** is disabled during display (non-interactive state) to prevent
move conflicts.

### Hand Card Visibility (FR-UI-2)

All player hand cards are displayed visually (never completely hidden), with
visibility rules based on game mode:

**In Mixed Game Modes (Human vs AI, AI vs Human):**

- **AI player**: Cards always shown as card backs (hidden strategy)
- **Human player**:
  - When active (their turn): Cards face-up and clickable
  - When non-active: Cards face-down (opponent's privacy)

**In Same-Type Game Modes (AI vs AI, Human vs Human):**

- **Active player**: Cards face-up and visible
- **Non-active player**: Cards face-down (hidden)
- Applies equally to both player types

**Key Features:**

- Card count badge always visible (e.g., "🖐️ 5 cards")
- Only face-up cards are clickable for selection
- Prevents accidentally selecting face-down cards
- Ensures fair play visibility across all game mode combinations

## 🎯 Architecture Highlights

- **Immutable GameState**: FSM-based state enables testability, undo/replay
  support
- **Event-Driven UI**: Observer pattern decouples game logic from presentation
- **Web Workers**: Async AI computation prevents UI blocking (1–10 sec
  configurable)
- **Config-Driven Design**: AI strategy, target score, house rules all
  configurable
- **Modular Core**: 9 independent game logic modules, each with clear
  responsibility
- **Two-Column Layout**: Game board (left) + condensed stats sidebar (right) for
  efficient screen utilization
- **Interactive Card Selection**: Both hand and table cards are clickable for
  intuitive capture workflow
- **Comprehensive UI Testing**: 25 regression tests catch UI issues early (card
  selection, layout, state transitions)
